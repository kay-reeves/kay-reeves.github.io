CSS: Visual Rules

Typography

Tag Name Description
<font-family> Font Family Property used to change the typeface of text on your page.
<font-weight> Font Weight Property controls how bold or thing the font is.
<font-style> Font Style Property used to set the style of the font, primarily to control whether the text appears in a normal, italic, or oblique face.
<text-transformation> Text Transform Property used to control the case of text, allowing for transformation to uppercase, lowercase, or capitalized forms without altering the original content.

Additional Notes

Font Family

  • Websafe fonts are a group of fonts supported across most browsers and are a good fall back font, if other fonts don't work
  • You can have up to four (4) fonts in a font stack. For example: font-family: Caslon, Georgia, 'Times New Roman', serif;.
  • It is recommended to use quotation when using a typeface with multiple words.

Font Weight

  • Values include bold, normal (default), lighter (one weight lighter), and bolder (one weight bolder).
  • Numerical values range from 1 (lightest) to 1000 (boldest) but it is common to use increments of 100. 400 is equivalent to normal and 700 is equivalent to bold.
  • Not all fonts can be assigned a numeric weight and not all numeric fonts are available to all font. Check the MDN for each assigned font.

Font Style

  • Normal: Displays the text in a normal, upright style (the default).
  • Italic: Displays the text in an italicized style, which typically uses a specially designed italic version of the font.
  • Oblique: Displays the text in an oblique style, which is often a programmatically slanted version of the normal font if a true italic version is not available. While similar to italic, oblique fonts are generally less common and may be rendered differently across browsers.

Text Layout

Tag Name Description
letter-spacing: value Letter Spacing Property controls the amount of horizontal space between characters within a block of text.
word-spacing: value word-spacing Property controls the amount of space between words in a text.
line-height: value; Line Height Property specifies the height of each line of text within an element.
text-align: value Text Align Property controls the horizontal alignment of inline-level content within a block-level element.

Additional Notes

Letter Spacing

  • You can specify a precise amount of space using units like px (pixels), em (relative to the font size of the element), rem (relative to the root element's font size), or pt (points).

Word Spacing

  • It allows for adjusting the default spacing to increase or decrease the visual separation between individual words within an element.
  • A specific length unit (e.g., px, em, rem, pt) can be used to define the exact amount of space to add or subtract between words. Negative values are permitted to reduce the spacing.

Line Height

  • Value can be a unitless number. This is a multiplier of the element's own font size and is generally the preferred method for better inheritance and responsiveness. For example, line-height: 1.5; means the line height will be 1.5 times the font size of the element.
  • Changing the font size will auto adjust the height if used.

Text Align

  • Left: Aligns content to the left edge of the container (default for left-to-right languages).
  • Right: Aligns content to the right edge of the container.
  • Center: Centers the content horizontally within the container.
  • Justify: Distributes space evenly between words to make lines of text flush with both the left and right edges, creating a clean, block-like appearance.
  • Start: Aligns content to the beginning of the text direction (e.g., left for left-to-right languages, right for right-to-left languages).
  • End: Aligns content to the end of the text direction (e.g., right for left-to-right languages, left for right-to-left languages).
